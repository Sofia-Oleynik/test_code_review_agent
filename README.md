# Задача: Прогнозирование оттока клиентов

## Описание:

Компания мобильной связи хочет снизить отток клиентов. Вам предоставили датасет с информацией о клиентах, их тарифах, использовании услуг и фактом ухода. Постройте модель машинного обучения, которая будет прогнозировать вероятность оттока клиента.

Цель: Создать прототип модели с Recall не менее 0.75 для класса "клиент уходит" (1), так как бизнесу важно найти как можно больше потенциально уходящих клиентов.

## Данные (telecom_churn.csv):

- customer_id - ID клиента
- tenure - сколько месяцев клиент с нами
- monthly_charges - ежемесячный платеж
- total_charges - общая сумма потраченных денег
- phone_service - есть ли телефонная услуга
- internet_service - тип интернет-услуги
- contract - тип контракта
- payment_method - способ оплаты
- churn - целевая переменная (0 - остался, 1 - ушел)

## Критерии оценки решения

1. Разведочный анализ данных (EDA) - 2 балла
2 балла: Полный анализ распределений, выбросов, корреляций с визуализацией. Понятные выводы по каждому этапу

1 балл: Базовый анализ без глубоких инсайтов или с пропущенными важными наблюдениями

0 баллов: EDA отсутствует или выполнен формально

2. Предобработка данных - 3 балла
3 балла: Корректная обработка всех типов данных, работа с пропусками, кодирование, масштабирование. Обоснование выбранных методов

2 балла: Обработка есть, но с ошибками или без обоснования

1 балл: Только базовая предобработка с критическими недочетами

0 баллов: Предобработка отсутствует

3. Построение и валидация модели - 3 балла
3 балла: Корректное разделение данных, несколько алгоритмов, настройка гиперпараметров, кросс-валидация

2 балла: Базовая настройка моделей без оптимизации или с методологическими ошибками

1 балл: Одна модель без валидации и настройки

0 баллов: Модель не построена

4. Оценка качества и интерпретация - 2 балла
2 балла: Анализ нескольких метрик, feature importance, бизнес-интерпретация результатов

1 балл: Только базовые метрики без интерпретации

0 баллов: Оценка качества отсутствует
